What is the purpose of software requirements specification documentation?

The purpose of the requirements document is to collect and establish the needs of the client, so that a solid plan can be put in place before production of the system. The document will provide a detailed overview of the software product, like its target audience, user interface, software and hardware requirements. It also helps the client, designers and developers gain a view at the finished product.

How are functional and non-functional requirements addressed in the documentation?

All functional and non functional requirements should be clearly listed on the requirements document, going into details for each function as to create a clear understanding for the development of the system.
